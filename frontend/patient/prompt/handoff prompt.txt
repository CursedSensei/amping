Context:
We are building "Project Amping," a gamified medical adherence app for TB patients. The app features an animated AI companion named "Gabby" and relies on gamification elements like streaks, XP, and badges. 

Design System & Assets Available:
 Theme Colors: `DarkNavy`, `LightBackground`, `CyanPrimary`, `RedPenalty`, `TextDark`, `TextMuted`, `GoldYellow`.
 Custom Components: An `AnimatedGabby` composable exists and should be used whenever Gabby is speaking or guiding the user. 
 Icons: Material Icons Extended is available. 

Task:
Generate Jetpack Compose screens and their corresponding ViewModels based strictly on the screen definitions provided in the CSV data below. 

Strict Architectural Directives:
1.  MVVM Pattern: Every screen must have a dedicated ViewModel. 
2.  UI State: Use a `data class` to represent the UI state. The ViewModel should expose this via `StateFlow` (`MutableStateFlow` internally).
3.  Stateless Views: The UI composables must be stateless. They should collect the `StateFlow` and pass user intents (clicks, typing) back up to the ViewModel via lambda functions. Do not put business logic in the composable.
4.  Navigation: Do not implement `NavHost` here. Instead, pass navigation events as lambda parameters to the main screen composables (e.g., `onNavigateNext: () -> Unit`).
5.  Obsolete Code: Completely ignore and discard any previous iterations of `OnboardingScreen.kt`. Treat the CSV data below as the absolute single source of truth for the new onboarding and app flow.
